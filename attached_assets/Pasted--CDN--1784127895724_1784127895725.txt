Ответим на ваши вопросы по порядку.

Взимается ли фиксированная плата за каждый CDN-ресурс (мы видим ~150 руб/ресурс)? Или оплата только за трафик?

Каждый CDN-ресурс оплачивается по модели тарификации: Ресурс × месяц, и его стоимость составляет 150 ₽ в месяц. Тарификация представлена на этой странице.

Есть ли плата за добавление secondary hostnames (дополнительных доменов) к существующему CDN-ресурсу?

Платить за отдельные доменные имена не нужно.

В Cloud CDN поддерживаются любые доменные имена, в том числе IDN с символами национальных алфавитов не из ASCII, например кириллическими. Такие имена можно указывать в исходном виде (пример.рф) или преобразовывать в Punycode с добавлением префиксов xn-- (xn--e1afmkfd.xn--p1ai).

При создании ресурса нужно указать хотя бы одно доменное имя. Первое имя является основным, его нельзя изменить после создания ресурса. Подробнее об этом пишем тут.

Какова квота на количество CDN-ресурсов в одном облаке/каталоге по умолчанию?

По умолчанию квота на количество CDN-ресурсов в одном облаке — 20 ресурсов. Все запросы на повышение квоты согласовываются индивидуально и зависят от точных условий и сценариев. Квоты и лимиты для Cloud CDN описаны здесь.

Можно ли добавить 10 000+ secondary hostnames к одному CDN-ресурсу через API?

Нет, так как такое количество превысит максимальный размер тела запроса при создании ресурса.

Передаёт ли CDN заголовок Host оригинального запроса на origin-сервер? (нам критично читать Host на origin для маршрутизации к нужному бакету)

Чтобы CDN-серверы отправляли корректные запросы к источникам, можно настроить значение HTTP-заголовка Host для этих запросов. Значение заголовка Host выбирается при создании ресурса. После этого настройку можно изменить вместе с другими основными настройками ресурса. Информация о заголовке Host есть на этой странице.

Поддерживает ли один CDN-ресурс несколько сертификатов из Yandex Certificate Manager одновременно (по одному на каждый кастомный домен через SNI)? Каков лимит на количество прикреплённых сертификатов к одному CDN-ресурсу?

На каждый CDN-ресурс можно установить только один сертификат: это может быть новый сертификат Let's Encrypt® или собственный.

Возможно ли автоматическое прикрепление нового сертификата через API без простоя CDN?

Вы можете настроить прикрепление автоматического нового сертификата к CDN-ресурсу с помощью поля custom_server_name в спецификации Terraform или с помощью флага--cert-manager-ssl-cert-id в спецификации YC CLI.

Отметим, что создание такого скрипта находится в зоне ответственности пользователя.

Поддерживается ли purge кеша по конкретному hostname/домену (не по всему CDN-ресурсу)?

Вы можете очищать кэш по конкретному домену, не затрагивая весь CDN-ресурс. Для частичной очистки кеша можно указывать пути к конкретным файлам и каталогам. Каждый путь должен начинаться с символа /.

Есть ли API для точечного purge по URL-паттернам?

Если для ресурса включена сегментация файлов, при выборочной очистке кеша конкретного файла нужно учитывать, что файл хранится на CDN-серверах по частям. Используйте один из вариантов:

добавьте символ подстановки * в конец пути к файлу — например, /image/foobar.png*;

укажите в пути конкретный диапазон байт сегмента с помощью суффикса @bytes=<начало>-<конец> — например, /image/foobar.png@bytes=0-10485759 для очистки первого сегмента размером 10 МБ.

Подробнее об этом пишем в нашей документации.

Очистить кэш вы можете через консоль управления, методом REST API purge для ресурса Cache или вызовом gRPC API CacheService/Purge. На очистку кеша может потребоваться до 15 минут. Подробнее об очистке кеша ресурса пишем на этой странице.
